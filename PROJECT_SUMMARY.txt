KEY FEATURES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. EXPRESS API - Quick Plotting
   ┌─────────────────────────────────────────┐
   │ perspicillum::plot(x, y)                │
   │   .title("Title")                       │
   │   .name("Series 1")                     │
   │   .color(0.2f, 0.6f, 1.0f)              │
   │   .width(2.0f)                          │
   │   .show();                              │
   └─────────────────────────────────────────┘

2. OOP API - Detailed Control
   ┌─────────────────────────────────────────┐
   │ perspicillum::Figure fig(2, 2);         │
   │ fig.subplot(1, 1).add_plot(             │
   │   std::make_unique<LinePlot>(x, y)      │
   │ );                                      │
   │ fig.show();                             │
   └─────────────────────────────────────────┘

3. DATA FLEXIBILITY
   ┌─────────────────────────────────────────┐
   │ std::vector<double> v = {1, 2, 3};      │
   │ Eigen::VectorXd ev = ...;               │
   │ std::span<const double> s = v;          │
   │                                         │
   │ plot(v).show();  // Works!              │
   │ plot(ev).show(); // Works!              │
   │ plot(s).show();  // Works!              │
   └─────────────────────────────────────────┘

4. GUI EMBEDDING
   ┌─────────────────────────────────────────┐
   │ ImGui::Begin("Plot");                   │
   │ figure.render();  // No window, renders │
   │ ImGui::End();     // in context         │
   └─────────────────────────────────────────┘

  $ make test
  $ ./tests/data_container_test

Run Examples:
  $ ./examples/quick_plot
  $ ./examples/figure_api
  $ ./examples/data_types

Integrate into Your Project:
  1. Add to CMakeLists.txt: add_subdirectory(perspicillum)
  2. Link library: target_link_libraries(your_app perspicillum)
  3. Include header: #include <perspicillum.hpp>
  4. Use C++20 standard
  5. Create plots and call .show()

✓ Header-First Design
  → Public API is mostly headers
  → Fast compilation for header-only parts
  → Implementation files only for heavy lifting

✓ Move Semantics
  → Efficient data transfer with std::move
  → Smart pointers throughout (no raw pointers)
  → Zero unnecessary copies

✓ Type Safety
  → C++20 concepts where beneficial
  → Template metaprogramming for static typing
  → Compile-time error checking

✓ Modularity
  → Clear separation of concerns
  → DataContainer abstraction
  → PlotBase interface for extensibility
  → WindowManager encapsulation

✓ Zero-Copy Data
  → std::span for existing buffers
  → Direct pointers to Eigen data
  → Efficient vector handling

✓ Plotly Compatibility
  → Familiar API from JavaScript Plotly.js
  → Normalized color values (0-1)
  → Similar method names and style
